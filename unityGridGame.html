Unity Engine Grid Game

A Grid-based game made with Unity Engine

This project was done to play around with the UI System of the Unity engine. The focus was to make a responsive UI that can auto adjust to any resolution. In order to do so, I decided to make a puzzle-grid game, where players can add dots to create the largest spanning path a.k.a. Hamiltonian Path possible.

Baisc Idea

The idea of the game is simple: to create the longest spanning path possible by connecting dots The porject utilizes URP (Universal Render Pipeline) for rendering graphics. However, graphics was not the priority of this project. I used TextMeshPro for all the menu UI. The main game UI was done using UI Anchors which allowed me to snap assets to any part of the screen. I also used URP post-processing to make it more aestheically pleasing.

I made this project with Unity 2019.4.40f1 LTS.

Gameplay

The game consists of Rows X Columns which can be modified from the "settings" option on the main menu. By default the game starts with at 5X5 grid. By clicking on any tile, you can place a 'dot'. Once 'n' number of dots are placed in adjacent rows or columns, a line automatically connects them.

I have tried running the game at different resolution and differenct aspect-ratios but thanks to UI anchors, the grids, lines and the dots itself maintain their position on the screen which was my main objective of the project.

I add particle effects and a bit of URP post processing such as simulating cloud rolling effect at the background as well as my own background score to complete the game.

UI

You can see on the image on the right, the rows and columns can be dynamically created. Currently there is a limit to the number of rows and columns that can be created to maintain the responsiveness of the grid itself. Based on my testing that number is a max of 25. Any bigger number causes the grid to overflow from the screen.

However, this is not a problem of the UI anchors. In order to accomodate more grid, the grid size itself shrinks, at which point the grid size becomes so small for the player to even click on the cells.

Retrospective

In short this project is more of an experimentation with the UI system rather than a full fledged game. The UI still needs some work and the game itself does not pose any challenge. The idea can be further expanded to create obstacles and add more responsiveness to the UI.
