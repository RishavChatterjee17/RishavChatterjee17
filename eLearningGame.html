E-Learning Management System

A Java servlet to create an E-Learning Online Platform to learn programming concepts, participate in quizes and post your questions/answers in the forum. This servlet was made using Java Enterprise Edition and JSP (Java Server Pages) was used to create the dynamic web application.

I chose to make a servlet for this project rather than a CGI (Common Gateway Interface) as Servlets have many advantages over CGI. Some of the advantages are better performance: as each request is assigned a thread instead of a process, hence, requests/responses are faster and it can handle more number of clients. JVM (Java Virtual Machine) also manages servlets so it is portable and robust. Thus we dont need to worry about memory management and memory leaks. Also servlets are more secure than CGIs.

Basic Idea

The idea of this project was to make a "Hackerrank"-like online platform for learning how to code, test your knowledge and have conversations in the public forum.

I started by creating a admin/user register and login. To do this I set up a SQL Database to store credentials as well as the content for the forums. I also set up Apache Tomcat to run the servlet and the JSPs.

From the Flow Diagram on the right we can see the entire scope of the project from the client side. The forums are logically separated into question and answer forums which seemed like a good idea then. Users could take tests, view the forum, post in the forum, view their profile to view their progress.

Structure

I wanted to have a realtime player progression feature like hackerrank based on how many courses a user completes and how many quizes they participate in. Thus I created a user profile page, where users could see their progress in realtime.

I made an admin and user registration, signin and logout page. The credentials for these were stored in the SQL Database. For adding a bit of security I implemented page sessions so that user cannot simply enter a URL path and skip singin or access someone else's account.

The admins were responsible for creating the study material and questions for quizes. Users could answer other user's questions on the forum and edit their responses. The admin had the rights to remove threads from the forum.

Features

One of the most important features of the application is to participate in online quizes and get score in realtime. The quizes were all in MCQ pattern for easier implementation of scoring. A score history was maintained to calculate overall average performance and increase profile progression.

Users could choose from a variety of courses to study. The study content consisted of static text stored in XML format and Youtube videos integrated into the web applications.

Retrospective

This was probably the first web application I created, hence the UI/UX is not ideal. None of the features were made with accessibility or ease of navigation in mind.

Another problem is the increased number of SQL queries we needed to run for getting forum data made the request/response slower. I could have stored frequently used data on the frontend thus reducing unecessary requests to the server.

Although I added quite some many features, none of the features are flushed out and could benefit from further polishing.
